Creating and Managing Gallery Experiences

{{ page.description }}

If you are looking to quickly get started creating and publishing a Gallery experience, check out one of these documents:

  • Step-by-Step: Creating and Publishing a Portal Experience
  • Step-by-Step: Creating and Publishing an In-Page Experience

When you open the Gallery module, the Experiences page will list all of the experiences that have been created.

If no experiences have been created, you will be prompted to create an experience. The Experiences page provides the ability to:

  • Delete experiences
  • Take experiences offline
  • Duplicate experiences
  • Edit experiences
  • Create experiences
  • Filter experiences
  • Search experiences

Click on a column header to sort the data. The column selector () can be used to select the columns to display on the page.

The following columns may be displayed.

  • Title - Title of the experience
  • Template - Template used by the experience. The type of experience is displayed below:
    • Portal - Site that supports a content library of any size with optional call-to-action and ad support
    • In-Page - Publish one or more videos and add interactivity and then add this layout to your site
    • Landing Page - Single-page site with lead form and optional call-to-action and ad support
    • Event - Site optimized for promoting and broadcasting live events
  • Template Status - Status of the template
    • Up to Date - The experience is using the latest version of the project template
    • Needs Update - A template update for the experience is available
    • Resume Update - Template update is in progress but hasn't been published
  • Updated - Date and time of the last experience update
  • Experience Status - Status of the experience
    • Online - The experience is online and can be accessed
    • Offline - The experience is offline and cannot be accessed
    • Unpublished Changes - The experience is online and can be accessed but changes have been made to the experience and have not been published

Clicking on a row will display the experience details on the right side of the page.

Creating experiences

To create a new experience, click New Experience. When you create a new experience, you'll start by choosing a template. The template controls the overall look and behavior. After selecting a template, you can customize the experience. For more information, see one of these topics:

  • Overview: Creating, Editing and Publishing Portal Experiences
  • Overview: Creating, Editing and Publishing In-Page Experiences

Duplicating experiences

Duplicating an experience can be useful when you need to create an experience in multiple languages or if you're making microsites from the look and feel of an existing experience. To duplicate an experience, click on a row and then click Duplicate.

Enter an Experience Title and Description and then click Save.

Editing experiences

To edit an experience, click on the experience Title. Editing an experience will open the Gallery Site Editor. For information on editing experiences, see one of these topics:

  • Using the Site Editor to Customize a Portal Experience
  • Customizing an In-Page Experience

Taking experiences offline

Taking an experience offline makes it no longer accessible. To take an experience offline, click on a row and then click Take Offline. To confirm that the experience should be taken offline, click Take Offline. To make an experience available again, the experience must be republished.

Deleting experiences

To delete an experience, click on a row and then click Delete. Confirm the deletion by clicking Delete.

Note: If your experience is configured to use a custom domain, you should remove the custom domain assigned to the experience and then re-publish the experience BEFORE deleting the experience.

Filtering experiences

A filter can be applied to the Experiences page to limit the types of experiences that are displayed. To apply a filter, select the type of experiences to display:

  • All - Display all experiences
  • Published - Display all published (online) experiences
  • Unpublished - Display all experiences that have been taken offline
  • Template Update Needed - Display all experiences that have template updates available
  • Landing Page - Display all Landing Page Experiences
  • Event - Display all event experiences
  • Portal - Display Portal Experiences
  • In-Page - Display all In-Page Experiences

Searching experiences

The Search box can be used to search through the list of experiences. When searching, the following properties will be searched:

  • Title
  • Description
  • ID
  • Template
  • URLs
