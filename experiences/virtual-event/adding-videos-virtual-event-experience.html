Adding Videos to a Virtual Event Experience

{{ page.description }}

Using the Gallery Collection Editor, you can select the videos to display on a Virtual Event Experience. Like other portal-type experiences, videos are organized into collections.

To add videos to a Virtual Event Experience, click Videos in the left navigation.

Video collection organization

The order of the collections will control how they appear on the virtual event homepage.

Note: For information on creating collections and adding videos to a collection, see Adding Videos to a Portal Experience.

The Virtual Event template has four main areas where videos are displayed as designated in the sample site:

  1. Featured collection - Typically used to feature the live or simulive content that will be displayed throughout the event. This should ideally be ordered based on its go-live time from earliest to latest for the best experience.
  2. Primary collection - Can include anything but is typically used to feature specific VOD breakouts or content you want to drive additional attention to
  3. Secondary collection - Can include anything but is typically used to feature specific VOD breakouts or content you want to drive additional attention to
  4. Tracks - Typically used for VOD content
Note: The Featured, Primary and Secondary collections support a maximum of 24 videos.
Note: To insert a title above the primary/secondary collection, add a static content object to the configuration file. For information, see Working with Static Content.

When ordering collections in Gallery, the collections should be ordered in the following manner:

  • Collection 1 - Featured collection
  • Collection 2 - Primary collection
  • Collection 3 - Secondary collection
  • Collection 4 - Tracks

Configuring event tracks

When creating the tracks, follow the guidelines below to make sure they appear correctly in the experience:

  • The collection names will appear in the virtual event template so use meaningful names
  • The tracks collection will be a nested collection, no videos will appear in this collection
  • The tracks collection should be the fourth collection
  • Create child collections under the tracks collection for each track
  • Do not use playlists when creating the collections
  • The position of the tracks section is specified in the configuration file and can be changed by editing the breakout_collection object in the configuration file

Configuring the tracks URL

Once the collections have been created, the configuration file needs to be updated so that the On-Demand Sessions link in the header will direct to the correct URL. The name used for the tracks collection (collection 4) will be used in the URL to access the tracks page. To ensure the URL is set correctly, confirm the following.

  1. Review the configuration file and locate the breakout_collection object.
  2. The collection parameter contains the URL of the tracks page. For example:
    
"breakout_collection": {
    "collection": "/category/videos/tracks",
    "insert_target": ".bc-on-demand-section",
    "insert_mode": "after"
},
  3. The last path in the collection URL should reflect the name of your tracks collection (collection 4). For example, if your tracks collection (collection 4) is named 'Sessions', the collection parameter would be updated to:
    
"collection": "/category/videos/sessions",
  4. To verify the configuration, click the On-Demand Sessions link in the header. The tracks session should appear. Clicking on a track should open the videos in a track. If you see a 404 error, there is a configuration issue.
