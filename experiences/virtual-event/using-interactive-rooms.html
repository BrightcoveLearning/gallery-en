Using Interactive Rooms

{{ page.description }}

Interactive rooms provide the ability to hold online meetings that support audio, video and screen sharing. Interactive rooms are created by adding a session with a Session Type of Interactive Room to a virtual event agenda.

Note: Interactive Rooms currently support up to 25 participants (speakers and attendees). If more than 25 participants attempt to enter a room, they will receive a message that the room is full.

When the session start date and time are reached, a JOIN SESSION button will appear on the agenda. Attendees can click the button to enter the interactive room.

To use interactive rooms, the Virtual Event Experience must be secured using SSO. The experience will inspect the user ID used to login. If the user ID matches an email address for one of the session speakers, that user will be presented with the speaker interface. All other users will be presented with the attendee interface.

Setting up audio and camera

When entering an interactive room, the settings page will appear allowing you to configure your audio and camera settings.

  • Camera - Select the camera to use; after selecting a camera, the camera preview image will update
  • Video quality - Select the video quality for the interactive room
  • Mic - Select the mic to use
  • Speaker - Select the speaker to use for audio output
  • Attendee settings - Allow attendees to enable their cameras and microphones
Note: The settings can also be configured from inside the interactive room interface once the conference has started.

After the settings are configured, click Ready to enter the interactive room.

Using the interactive room interface

The interface provides a set of controls for controlling audio and video for the meeting.

Enabling attendee cameras and mics

Presenters have the ability to enable and disable mics and cameras for attendees. When the presenter disables cameras and mics, all attendees will be muted and cameras will not be displayed. When the meeting begins, mics and cameras are enabled by default.

Viewing the meeting attendee list

Click the people icon ( ) to toggle the display of meeting attendees. The attendee list will display on the right side of the page. If the attendee camera is enabled, their image will be displayed.

Using chat

If chat has been enabled for the session, click the chat icon ( ) to open the chat panel.

Viewing related content

The presenter may have provided links to related content. Click the materials icon ( ) to view the download links that have been provided. The materials icon will only appear if related content has been added to the session.

Changing camera and mic settings

Click the settings icon ( ) to open the settings dialog. The settings page can be used to configure your audio and camera settings.

Note: Video will be turned off when the settings dialog is opened. Video will restart when the dialog is closed.

Ending and leaving the session

The End session button can be used by any speaker to end the session. The Leave session button can be used by any attendee to leave the session.

Speaker and attendee controls

A set of icons will appear at bottom of the screen allowing attendees to control their mic, camera, screen share and raise their hand. Note that the raise hand icon is only available to attendees.

Enabling your mic

If the speaker has enabled mic access, you can turn on your mic by clicking the mic icon. Your mic can only be activated if you have raised your hand and you are in the queue.

Attendees who have raised their hand will be displayed at the top of the queue in the attendee listing.

Speakers can enable audio and camera sharing and remove attendees from the queue by hovering over the attendee and clicking the appropriate icon(s).

Known issues/behaviors

Video and audio settings

Video and audio settings are not persisted between sessions. You must select new settings everytime a new session is started.

Unable to select an audio output device in some browsers

Firefox and Safari have known issues disallowing them from listing audio output devices on these browsers. While clients can continue the meeting using the default device, they will not be able to select devices in meetings. Chrome and Firefox on iOS also have the same issue. Android Chrome has a known issue switching between Bluetooth audio output devices. While clients can continue the meeting using the default device, there is a bug related to switching to a Bluetooth audio output.

Presenters sharing videos

Attendees will have no audio from videos that are screen shared by a presenter. This is the expected behavior.

If you turn on your camera and put the browser in background, others in the meeting will see black tile in Safari/Chrome/Firefox in iOS and frozen tile in Chrome in Android

This is the default behavior that is specific to each browser. In Android, the video stream is muted when in background and thus, the video will show the last frame. Note that for Android, if an attendee joins later after the video stream is in background, it will show as a blank tile since the last frame is blank.
