Adding Interactivity to a Breakout Room

{{ page.description }}

Pigeonhole can be used to easily add event based interactivity (live Q&A, polls, surveys, etc.) to a breakout room. Breakout rooms are interactive sessions that can be added to a Virtual Event experience. Typically, a new Pigeonhole is created for the live event and then sessions can be created for each video or breakout room you want to add Pigeonhole functionality to. Once the sessions are created, a Pigeonhole session ID and passcode are added to each breakout session in the live event agenda.

Note: The process to configure Pigeonhole for VOD videos in an agenda is slightly different. For information, see Integrating Pigeonhole with a Virtual Event Experience.

Creating a new Pigeonhole event and session

You should create a new Pigeonhole for your live event and then add sessions for each video that will have Pigeonhole functionality added to it.

  1. Log in to your Pigeonhole account.
  2. Click + Add Pigeonhole and click your account to use.
  3. Enter an Event name and Passcode.
  4. Enter Start and End dates for the event. These dates control how long the Pigeonhole will be active.
  5. Select a Timezone and Custom Branding option (if applicable).
  6. Click Continue.
  7. Now that the event is created the Agenda item will be selected in the left navigation. Click Add session and click Create new.
  8. Select a Session Type of Q&A (other session types can also be used).
  9. Enter a Session name.
  10. Enter Start and End dates and times.
  11. Configure the Q&A Settings as appropriate.
  12. Click Add Q&A and confirm the session was created.
  13. Create other sessions as needed.

Generating the Pigeonhole embed code

In order for a Pigeonhole to display next to a video, iframe embed code will be added to video using a Video Cloud custom field. The format of the code will be:

<iframe src="Pigeonhole URL"></iframe>

To get the value for the pigeonhole URL, follow these steps.

  1. In Pigeonhole, click Run your event in the left navigation.
  2. Click the link in the Invite your attendees section.
  3. Confirm that a new browser tab with a list of all sessions opens.
  4. Click the enter link for the appropriate session, for example, Enter this Q&A. Note that the link will change based upon the session type.
  5. Confirm that the Pigeonhole opens in a new browser tab. The Pigeonhole URL is displayed in the browser address bar.

The iframe tag with the Pigeonhole URL will be added to the appropriate video.

Configuring event settings

Settings for the Pigeonhole event must also be configured so the SSO information can be used to create a user profile.

  1. In the left navigation, click More settings.
  2. Click the Attendees tab.
  3. Check the Allow creating attendee profiles through embed option.
  4. Scroll down to the Registrant Profiles section on the page.
  5. Check the Enable registrant profile for this event option.

Creating a custom field

In order for the Virtual Event template to know which Pigeonhole to display for a video, a custom field is used. To create a custom field, follow these steps.

  1. Log in to Video Cloud Studio.
  2. In the navigation header, click ADMIN and then Video Fields. Note that you must be an account administrator to access to the ADMIN menu.
  3. Create a new custom field named Chat Embed.
  4. Confirm that the internal name is chat_embed.
  5. Set the Type to be Text.
  6. Save the custom field.

For information on creating custom fields in Video Cloud, see Creating Custom Metadata Fields.

Adding a Pigeonhole to a video

To have a Pigeonhole appear next to a video in the Virtual Event template, follow these steps.

  1. Open the Media module.
  2. Click on a video to open the video properties.
  3. Locate the CUSTOM FIELDS section and click Edit.
  4. Paste the Pigeonhole embed code into the Chat Embed custom field.
  5. Click Save.

Passing user SSO information to Pigeonhole

If your Virtual Event Experience is secured using SSO, it is possible to pass user SSO information to Pigeonhole so that users can be tracked inside of Pigeonhole. The following sections walk through the settings and steps that are needed to pass SSO information to Pigeonhole.

Map SSO attributes

You need to ensure that the correct SSO attributes are mapped so they can be passed to your experience and then to Pigeonhole. The integration Pigeonhole looks for the following values:

  • fname - User's name
  • nameID - User's email address

Expose attributes in the Access Control Profile

Ensure that the Access Control Profile used to secure the experience has the Expose all attributes option checked. If the SSO information is not available, users will appear as anonymous in Pigeonhole.

  1. Open the Gallery module.
  2. Click Settings in the header.
  3. In the left navigation, click Access Control Profiles.
  4. Click the edit icon for the profile being used to secure the experience.
  5. Ensure that the Expose all attributes setting is checked.

For information on configuring single sign-on, see Configuring Access Control Profiles. If everything is configured correctly, the user info will display in Pigeonhole when the viewer logs into the experience.
