Adding the Configuration File to the Experience

{{ page.description }}

The Virtual Event Experience requires a configuration file be included with the site. The configuration file includes site settings, layout options, event schedule, speakers and other content that will be displayed. After creating a new Virtual Event Experience, adding the configuration file is the first thing that should be done

To add the configuration file to a Virtual Event Experience, follow these steps.

  1. Go to the following link:
    https://virtual-events-experience.s3.amazonaws.com/v1/config.txt
  2. Copy the content to the clipboard.
  3. Open the Gallery module.
  4. Edit the experience.
  5. Click Appearance and Behavior > Header and Footer in the left navigation.
  6. Paste the configuration file into the Custom Header HTML field.
  7. Click Save.

Make sure you republish the experience so the changes appear.

Updating the policyKey and assetBaseUrl values

In order for the countdown timer to display on simulive videos, the policyKey value in the configuration file needs to be updated with a search-enabled policy key. The value that is supplied will not work for your account. Failure to do this will result in no countdown being displayed with the videos. For steps to get a search-enabled policy key, see the developer document Code Sample: Generate a Policy Key. After the new policy key is created, update the policyKey value in the configuration file.

The assetBaseUrl value path is used to load files used by the template. This can be left alone in most cases unless a high degree of customization is being performed. The path value that is there should be removed.

After the policyKey and assetBaseUrl values have been updated in the configuration file, republish the experience.
